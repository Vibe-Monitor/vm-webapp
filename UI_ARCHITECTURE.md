## 11. Key Interactions & Transitions

### Sidebar Collapse/Expand
- **Trigger**: Click on expand button (visible on hover when collapsed)
- **Animation**: Width transitions from 56px to 280px over 200ms ease-out
- **State persists**: Use localStorage to remember preference
- **Keyboard shortcut**: `Cmd/Ctrl + \` to toggle

### Space Switching
1. Click on space header in sidebar
2. Dropdown appears with animation (fade + slide down, 150ms)
3. Click on different space
4. Loading state shown briefly
5. API calls to fetch new space data (environments, services, etc.)
6. UI updates: sidebar closes dropdown
7. Route stays same, context changes
8. Last visited workspace updated via API

### Chat Feedback
1. User clicks thumbs up/down on bot response
2. Button shows loading state (spinner)
3. API call to submit feedback
4. Success: button changes to "filled" state with accent color
5. User can change feedback (toggles between states)
6. Optimistic update with rollback on error

### Settings Navigation
- Settings items in sidebar navigate to respective routes
- Active item highlighted with accent background
- Content area transitions with fade effect (100ms)
- Scroll position preserved when switching back

### Create Workspace Flow
1. Click "Create New Space" in space switcher
2. Modal opens with fade + scale animation
3. Select space type (Personal/Team)
4. Tooltip shows on type selection with details
5. Enter space name
6. Click Create â†’ loading state
7. On success: modal closes, space added to list, auto-switch to new space
8. On error: show error message in modal

### Invite Member Flow
1. Click "Invite Member" button
2. Modal opens
3. Enter email, select role
4. Helper text explains role permissions
5. Click Send â†’ loading state
6. On success: modal closes, invitation added to pending list, toast notification
7. On error: show error in modal

### Delete Account Flow
1. Click "Delete Account" button
2. API call to get deletion preview
3. Modal shows with preview data:
   - Blocking workspaces (if any) - delete button disabled
   - Workspaces to be deleted
   - Workspaces to leave
4. User types "DELETE" and password (if credential account)
5. Delete button enables
6. Click Delete â†’ loading state
7. On success: redirect to landing page, clear all state
8. On error: show error in modal

## 14. Accessibility Considerations

### Keyboard Navigation
- All interactive elements focusable with Tab
- Sidebar items navigable with arrow keys
- Escape closes modals and dropdowns
- Enter/Space activates buttons
- Focus trap in modals

### ARIA Labels
- Sidebar toggle: `aria-label="Toggle sidebar"`
- Space switcher: `aria-haspopup="listbox"`, `aria-expanded`
- Chat input: `aria-label="Type your message"`
- Status badges: `aria-label="Status: Connected"`

### Color Contrast
- All text meets WCAG AA (4.5:1 for normal, 3:1 for large)
- Error states have icon + color (not color alone)
- Focus rings visible (2px solid blue)

### Screen Reader Support
- Live regions for chat messages (`aria-live="polite"`)
- Loading states announced (`aria-busy="true"`)
- Error messages linked to inputs (`aria-describedby`)
